## Future Module Contract

Every future module must define:

- manifest
- dependencies
- database migrations
- public/admin routes
- API routes
- permissions
- event subscriptions
- seed data
- tests

## Candidate Modules

### Loyalty Module

Features:

- points
- rewards
- customer tiers
- birthday offers

### Memberships Module

Features:

- recurring plans
- member portal
- credits/packages
- Stripe subscriptions

### Intake Forms Module

Features:

- pre-appointment forms
- consent capture
- clinic mode support
- timeline integration

### Reviews Module

Features:

- post-visit review requests
- testimonial approval
- public display blocks

### Inventory Module

Features:

- stock tracking
- low-stock alerts
- supplier records
- restaurant/salon use cases

### Optional AI Assistant Module

Not part of core scope.

Possible future features:

- draft responses
- summarize customer timeline
- suggest booking follow-up
- generate marketing copy

Must be implemented as a separately enableable module.
