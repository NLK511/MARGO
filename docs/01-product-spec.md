## Product Overview

A white-label business webapp platform that lets an operator quickly deploy polished, branded websites with optional booking and CRM capabilities.

Target businesses:

- restaurants
- clinics
- dentists
- physiotherapists
- salons
- spas
- gyms
- wellness studios
- consultants
- repair/service companies

## User Personas

### Platform Owner

Runs the SaaS/productized agency and provisions tenants.

Needs:

- tenant creation
- plan/module enablement
- billing visibility
- global configuration
- support/debug tools

### Tenant Owner

Owns a business using the app.

Needs:

- manage brand/theme
- manage pages
- manage services
- view bookings/customers
- invite staff
- configure integrations

### Staff / Front Desk

Handles daily operations.

Needs:

- view calendar
- create/edit/cancel bookings
- check customers in
- update customer records
- take notes

### Provider / Clinician / Server / Resource Owner

Performs the service.

Needs:

- view assigned appointments
- add internal notes
- mark completion/no-show
- limited CRM access

### Customer / Patient / Guest

Uses public-facing flows.

Needs:

- browse business information
- book/reserve appointments
- manage/cancel/reschedule booking
- receive confirmations/reminders

## Module Packages

### Package A: Front Page Only

Includes:

- tenant resolution
- theme engine
- public page renderer
- page sections
- contact/lead forms
- SEO metadata

Excludes:

- booking engine
- CRM dashboard
- payments

### Package B: Front Page + Booking

Includes Package A plus:

- services/resources
- availability engine
- public booking flow
- booking management links
- notification templates
- optional payments/deposits

### Package C: Booking + CRM

Includes:

- booking staff dashboard
- customer/patient profiles
- notes/tags/timeline
- role-based access

May exclude:

- full marketing page builder if deployed as operational-only app

### Package D: Full Suite

Includes all current modules.

## Product Principles

- Modular first
- Business-outcome oriented
- Configurable rather than forked
- Reusable across verticals
- Aesthetic flexibility without breaking accessibility
- Simple MVP path, extensible long-term architecture
